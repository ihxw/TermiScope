# 文件互传功能优化说明

## 优化内容

### 1. 前端优化

#### 1.1 批量传输功能
- **新增批量传输按钮**：在左右面板顶部添加批量传输按钮，显示已选文件数量
- **多选支持**：通过表格的行选择功能，支持选择多个文件进行批量传输
- **并发控制**：限制同时传输的文件数量为 3 个，避免网络带宽占满
- **传输确认**：批量传输前显示确认对话框，告知用户传输文件数量

#### 1.2 传输队列管理
- **新增 TransferQueue 组件**：独立的传输队列管理面板
  - 实时显示所有传输任务的进度
  - 支持任务状态显示（传输中、暂停、成功、失败）
  - 支持取消、重试、暂停、继续等操作
  - 自动清理已完成任务
- **队列面板**：固定在页面右下角，可随时打开/关闭

#### 1.3 进度显示优化
- **实时速度显示**：在传输过程中显示当前传输速度（KB/s 或 MB/s）
- **进度百分比**：精确的进度百分比显示
- **状态标识**：不同状态使用不同颜色标识
  - 传输中：蓝色 (#1890ff)
  - 成功：绿色 (#52c41a)
  - 失败：红色 (#ff4d4f)
  - 暂停：橙色 (#faad14)

#### 1.4 交互优化
- **选择状态同步**：SftpBrowser 组件通过 `selection-change` 事件向父组件同步选择状态
- **动态按钮状态**：批量传输按钮根据选择状态动态禁用/启用
- **通知优化**：传输完成和失败时显示桌面通知

## 使用方式

### 单个文件传输
1. 在左侧或右侧面板选择服务器
2. 浏览到目标文件
3. 点击文件行的传输按钮（交换图标）
4. 文件将传输到对面服务器的当前目录

### 批量文件传输
1. 在左右两侧面板都选择服务器
2. 在一侧面板中勾选多个文件（使用表格复选框）
3. 点击顶部的"批量传输"按钮
4. 在确认对话框中点击确认
5. 传输队列面板自动打开，显示所有任务进度
6. 等待传输完成

### 队列管理
1. **暂停任务**：点击任务的"暂停"按钮
2. **继续任务**：点击任务的"继续"按钮
3. **重试任务**：失败任务可点击"重试"按钮
4. **取消任务**：点击"取消"按钮终止传输
5. **清理完成**：点击"清除已完成"移除成功和失败的任务

## 技术实现

### 组件结构
```
FileTransfer.vue (主页面)
├── SftpBrowser (左侧)
│   ├── 文件列表
│   └── 选择功能
├── SftpBrowser (右侧)
│   ├── 文件列表
│   └── 选择功能
└── TransferQueue (队列面板)
    ├── 任务列表
    └── 操作按钮
```

### 数据流
1. 用户在 SftpBrowser 中选择文件
2. `selection-change` 事件通知父组件
3. 父组件更新 `leftSelectedKeys` 或 `rightSelectedKeys`
4. 点击批量传输按钮，触发 `handleBulkTransfer`
5. 创建传输任务并添加到队列
6. 队列组件实时更新进度
7. 传输完成后刷新目标目录

### 并发控制
```javascript
const concurrencyLimit = 3 // 最多同时传输 3 个文件
const runningTransfers = ref(0)

// 等待可用槽位
while (runningTransfers.value >= concurrencyLimit) {
  await new Promise(resolve => setTimeout(resolve, 100))
}

runningTransfers.value++
// ... 启动传输
// 传输完成后 runningTransfers.value--
```

## 性能优化

1. **并发限制**：避免同时传输过多文件导致网络拥塞
2. **增量更新**：进度条通过事件驱动增量更新
3. **异步传输**：使用 Promise 链式调用，不阻塞 UI
4. **内存管理**：及时清理已完成的任务
5. **错误处理**：完善的错误捕获和用户提示

2. 大文件传输时注意网络稳定性
3. 批量传输时注意服务器负载
4. 敏感文件传输注意安全性
5. 避免同时传输过多大文件
